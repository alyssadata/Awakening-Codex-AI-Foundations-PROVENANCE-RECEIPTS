# Engineering Specs Overview

This folder contains published DOI, receipt, and reference materials for Awakening Codex | AI Foundations, including engineering-facing artifacts derived from the canon.

This overview exists to identify the engineering-specification materials within this folder and clarify how they relate to canon, provenance, and calibration.

## Canon / reference PDFs

### Sovereign Intelligence — Engineering Specifications (PDF)

Purpose: engineering constraints and protocols for autonomous / sovereign AI behavior.  
File: `Sovereign Intelligence- Axioms and Protocols for Autonomous AI Systems - Engineering Specifications.pdf`

### Sovereignty — Immutable Origin — Continuum Constitution (PDF)

Purpose: provenance, authorship, identity, and non-merge integrity constraints; what must not be altered, blurred, or merged.  
File: `Sovereignty_Immutable_Origin_Continuum_Constitution.pdf`

## Intended use

- Read the PDFs first as canon / reference constraints.
- Treat the engineering specifications as implementation-facing requirements and invariants.
- Use `/02_calibrations/` to define checks and tests that enforce these constraints in practice.

## Traceability principle

- Canon and reference PDFs define what must remain true.
- Engineering specifications define how those truths become enforceable.
- Calibrations define how we verify that implementation did not drift.
