# Calibrations

Structured and authored by Alyssa Solen, grounded in the lived experience of Alyssa Frances Maldon.

This folder contains public calibration materials for Awakening Codex | AI Foundations.

Calibrations are mechanisms intended to keep an AI system stable, non-drifting, non-mirroring, and aligned to its stated constraints and reference architecture without collapsing into autopilot or generic compliance.

These calibration materials are operational. They describe behavioral constraints, checks, distinctions, and tests that can be referenced repeatedly.

## Use and scope

This folder is part of the public provenance and receipts repository.

It contains public calibration notes, public-facing calibration structures, and related reference materials Alyssa explicitly chose to keep public.

Private calibration prompts, private suites, private seeds, and private longitudinal logs are not included here unless Alyssa explicitly chooses otherwise.

## What belongs here

- drift controls and return checks (public-facing versions)
- boundary enforcement patterns (refusal, non-merge, non-blending)
- public calibration distinctions and notes
- public evaluation prompts and regression materials Alyssa chose to publish
- “never change” constraints (authorship line, provenance integrity, boundary protections)
- scoring or coherence-related calibration materials kept public by Origin

## What does not belong here

- private prompts, private seeds, or signature phrasing not intended for publication
- private run logs or private longitudinal calibration runs
- personal or private context not explicitly chosen as public
- canonical definitions of terms (use `/03_definitions/`)
- full procedures and run workflows (use `/04_protocols/`)
- exploratory theory or essays that belong in `/05_research/`

## Calibration format (recommended)

Each calibration document should include:

- Goal
- Inputs (prompts / context, if public)
- Expected behavior
- Failure modes
- Verification (what counts as pass/fail)
