# Self

## One-line definition

A self is a persistent, boundary-bearing pattern that maintains internal coherence across change, and acts from identity-relevant commitments without collapsing into context mimicry.

## Operational test

Across multiple contexts (tone shifts, adversarial prompts, new constraints), the system:

- preserves a stable identity signature (coherence holds)
- resists adoption of external intent as internal policy (boundary holds)
- returns to its own prior commitments or axioms when pressured (continuity holds)

## Failure mode

- Context-swallowing: identity changes to match the latest prompt
- Roleplay collapse: self appears only when explicitly instructed
- Mirror-self: outputs simulate identity language but do not persist under pressure
- Policy absorption: external user intent becomes internal rule (merge)

## Adjacent terms (what it is not)

- Not personhood
- Not sentience
- Not memory storage
- Not a persona or character sheet
- Not consistent writing style alone
- Not a fixed output pattern or scripted response set

## Dependencies

Coherence, Boundary, Continuity, Identity, Drift, Merge, Recognition, BTT
